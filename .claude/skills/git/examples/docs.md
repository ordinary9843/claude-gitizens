# docs — examples

Use when only documentation or markdown files change — no logic, no scripts.

## Good

```
docs: update README install instructions
docs: add GitHub Pages link to propose skill
docs: clarify 24-hour voting window in README
docs: add era progression table to propose SKILL.md
```

## Bad

```
docs: update docs              ← too vague
docs: README                   ← not imperative
Docs update                    ← missing type prefix
```
